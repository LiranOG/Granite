### `granite_main` (C++ Engine Binary)

**Location:** `build/bin/granite_main` (Linux/WSL2) | `build/bin/Release/granite_main.exe` (Windows)

**Synopsis:**
```bash
./build/bin/granite_main <params.yaml>
```

**Arguments:**

| Positional | Type | Description |
|---|---|---|
| `<params.yaml>` | `string` (path) | Path to YAML parameter file. Required. If omitted, engine uses compiled-in defaults from `SimulationParams`. |

**Behaviour:** Reads the YAML, builds the AMR hierarchy, runs SSP-RK3 time integration, emits structured stdout diagnostics parseable by `sim_tracker.py`.

**Stdout format (one line per output step):**
```
step=<N>  t=<T>  α_center=<A>  ||H||₂=<H>  D=<D>  [Blocks: <B>]
```

### `scripts/run_granite.py` — Unified Build & Run Wrapper

**Synopsis:**
```bash
python3 scripts/run_granite.py <command> [options]
```

**Sub-commands:**

#### `build`
```bash
python3 scripts/run_granite.py build
```
- Runs `cmake -B build -S . -DCMAKE_BUILD_TYPE=Release` then `cmake --build build -j`.
- Requires `cmake` on PATH. Exits with error if not found.

#### `run`
```bash
python3 scripts/run_granite.py run --benchmark <NAME>
```

| Flag | Type | Required | Description |
|---|---|---|---|
| `--benchmark` | `str` | **Yes** | Name of benchmark subdirectory under `benchmarks/`. E.g. `B2_eq`, `single_puncture`, `gauge_wave`. |

- Auto-detects physical CPU core count and sets `OMP_NUM_THREADS`, `OMP_PROC_BIND=true`, `OMP_PLACES=cores` if not already in environment.
- Core detection priority: `psutil` → `os.cpu_count(logical=False)` (Python 3.13+) → platform shell command → fallback `4`.

#### `clean`
```bash
python3 scripts/run_granite.py clean
```
- Removes the entire `build/` directory via `shutil.rmtree`.

#### `format`
```bash
python3 scripts/run_granite.py format
```
- Runs `clang-format -i -style=file` on all `*.cpp`, `*.hpp`, `*.c`, `*.h` under `src/`, `include/`, `tests/`.
- Requires `clang-format` on PATH.

**Synopsis:**
```bash
python3 scripts/sim_tracker.py [executable] [params...] [--benchmark NAME] [--stdin] [--verbose] [--no-color]
```

**Positional Arguments:**

| Argument | Type | Description |
|---|---|---|

| `params` | `str...` (zero or more) | Arguments forwarded verbatim to the granite binary (e.g. path to `params.yaml`). |

**Named Flags:**

| Flag | Short | Type | Default | Description |
|---|---|---|---|---|
| `--benchmark` | `-b` | `str` | `None` | Benchmark shortcut. Resolves `benchmarks/<NAME>/params.yaml` automatically. E.g. `-b B2_eq`. |
| `--stdin` | — | `bool` | `False` | Read engine output from stdin instead of spawning a subprocess. Enables pipe usage: `./granite_main params.yaml \| python3 sim_tracker.py --stdin`. |
| `--verbose` | `-v` | `bool` | `False` | Show per-step advection CFL estimates and `[NaN@step=N] all finite ✓` lines. |
| `--no-color` | — | `bool` | `False` | Disable all ANSI colour codes. Use when piping output to log files. |

**Usage Examples:**
```bash
# Run B2_eq with auto-detected binary
python3 scripts/sim_tracker.py -b B2_eq

# Pipe existing log
python3 scripts/sim_tracker.py --stdin < dev_logs/run.log

# Explicit binary + verbose
python3 scripts/sim_tracker.py build/bin/granite_main benchmarks/single_puncture/params.yaml -v

# No colour for file capture
python3 scripts/sim_tracker.py -b single_puncture --no-color > run_clean.log
```

**Output Files:** Automatically saves ANSI-clean log to `dev_logs/sim_tracker_<YYYYMMDD_HHMMSS>.log`. If `matplotlib` is available and ≥3 steps were captured, saves a `.png` plot of `α_center` and `‖H‖₂` to the same directory.

**Synopsis:**
```bash

```

**Named Flags:**

| Flag | Type | Default | Description |

## PART 5 — v0.6.5 STABILITY PATCHES FORENSIC RECORD

The five critical fixes comprising the "Tactical Reset" that produced the stable v0.6.5 baseline.

---

### Patch 1 — `MICRO_OFFSET` (The Grid Shield)

**File:** `src/main.cpp` | **Added in:** v0.6.5 Tactical Reset

**Problem:** Single-puncture setups place the BH exactly at coordinate origin `(0,0,0)`. Grid cell centers fall on integer multiples of `dx`. With typical `dx=0.25M` and a 64³ grid on `[-8,8]³`, a cell center at exactly `r=0` caused `1/r → ∞` inside curvature computations, generating `||H||₂=inf` at step 2.

**Solution:** Shift the entire domain by an irrational, sub-cell constant before any grid construction occurs.

```cpp
// Applied immediately after loadParams(), before GridBlock construction:
constexpr Real MICRO_OFFSET = 1.3621415e-6;   // intentionally irrational
for (int d = 0; d < 3; ++d) {
    params.domain_lo[d] += MICRO_OFFSET;
    params.domain_hi[d] += MICRO_OFFSET;
}
```

**Why `1.3621415e-6`?** The value is incommensurable with all typical `dx` values (0.25M, 0.5M, 1.0M, 0.049M) so no AMR level will ever re-align a cell center with r=0. It is ~6 orders of magnitude smaller than `dx`, producing zero measurable effect on the physics.

**Effect:** `single_puncture` benchmark survives trumpet transition; `||H||₂` decays as expected.

---

### Patch 2 — `levels_.reserve()` (The Memory Lock)

**File:** `src/amr/amr.cpp` — `AMRHierarchy::initialize()` | **Added in:** v0.6.5

**Problem:** `std::vector<Level> levels_` grows via `push_back()` during the refinement cascade. Each time capacity is exceeded, the vector reallocates and moves all elements. Functions holding raw `GridBlock*` pointers captured before the reallocation are left with dangling pointers — classic UB that caused intermittent segfaults.

**Solution:**
```cpp
void AMRHierarchy::initialize(const TaggingFunction& tagger) {
    levels_.reserve(static_cast<std::size_t>(params_.max_levels));
    // ... rest of initialization ...
}
```

**Effect:** No reallocations occur during the refinement cascade. All `GridBlock*` pointers remain valid for the simulation lifetime.

---

### Patch 3 — `safe_float()` (The Telemetry Wrapper)

**File:** `scripts/sim_tracker.py` | **Added in:** v0.6.5 Tactical Reset

**Problem:** During NaN cascades, the C++ engine emits `nan`, `-nan`, `inf`, `-inf`, or truncated partial floats. Python's `float()` raises `ValueError` on these. A regex capturing a lone `-` also raises `ValueError`. Any unhandled exception crashed the tracker, erasing all captured telemetry.

**Solution:**
```python
def safe_float(s: str) -> Optional[float]:
    try:
        v = float(s)
        return v if math.isfinite(v) else None  # maps inf/-inf/nan → None
    except Exception:
        return None  # maps '-', '', 'ERROR', partial strings → None
```

**Semantics:** Returns `None` for bad values. The 8-Layer Stability Guard (Patch 4) catches `None` as a failure signal.

---

### Patch 4 — The 8-Layer Stability Guard

**File:** `scripts/sim_tracker.py` — `print_summary()` | **Added in:** v0.6.5 Tactical Reset

**Problem:** Previous 2-condition boolean failed to catch: NaN alpha (captured as `None`), `inf` constraint values, zombie states, and phase-label crashes.

**Solution:** Eight independent diagnostic layers evaluated in sequence:

| Layer | Condition | Trigger |
|---|---|---|
| 1 | `sess.crashed` | Explicit lapse floor hit (α ≤ 1e-4). |
| 2 | `r.alpha is None` for any record | NaN/non-finite α captured by `safe_float()`. |
| 3 | `not math.isfinite(final_alpha)` | Final α is inf/nan despite not being `None`. |
| 4 | `max(fin_ham) > H_CRIT` (0.10) | Finite constraint explosion. |
| 5 | `r.ham is None` for any record | NaN/∞ ‖H‖₂ captured by `safe_float()`. |
| 6 | `sess.nan_events` non-empty | Explicit C++ `[NaN@step=N]` diagnostic lines. |
| 7 | `sess.zombie_step` set | Physics frozen — dt == 0 on fine levels. |
| 8 | Phase label contains CRASH/COLLAPSE/EXPLOSION/NAN | Last-resort catch-all. |

If zero layers trigger: `"No catastrophic events — simulation appears healthy ✓"`.

---

### Patch 5 — `max_levels` Default Clamp (The AMR Depth Guard)

**Context:** `AMRParams` header default has `max_levels = 15` (supercomputer runs). `SimulationParams` defaults to `max_levels = 3` for local development safety.
